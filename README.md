Português

O Codeflix é uma plataforma completa de streaming inspirada em soluções como Netflix e Amazon Prime Video. Este projeto faz parte do curso Full Cycle e tem como objetivo praticar conceitos avançados de arquitetura distribuída, microservices, mensageria, comunicação orientada a eventos, autenticação centralizada e escalabilidade. A aplicação realiza a gestão completa do catálogo de vídeos, incluindo cadastro e administração de categorias, gêneros e elenco, além de um pipeline automatizado de encoding para conversão dos vídeos para MPEG-DASH. O sistema é composto por múltiplos serviços independentes que se comunicam por HTTP, mensageria e streaming de eventos, adotando conceitos reais utilizados em plataformas modernas de streaming.

O catálogo administrativo conta com um frontend em React, backend próprio e banco de dados MySQL, permitindo CRUD completo e upload de vídeos RAW. O pipeline de encoding é desenvolvido em Go, consome eventos via RabbitMQ e envia os vídeos processados para buckets na Google Cloud Platform. Para o catálogo público, existe um frontend React voltado aos assinantes e uma API pública, com Elasticsearch garantindo buscas rápidas, enquanto Kafka, Kafka Connect e Debezium realizam replicação de dados em tempo real. A gestão de assinaturas é feita por um microserviço dedicado com Postgres e autenticação centralizada através do Keycloak, utilizando JWT para controle de acesso seguro. Toda a arquitetura é documentada utilizando PlantUML com o C4 Model, representando usuários, contexto do sistema, serviços, comunicação, bancos de dados e brokers. O projeto utiliza tecnologias como React, Go, PHP(Laravel), Keycloak, RabbitMQ, Apache Kafka, Kafka Connect, Debezium, Elasticsearch, MySQL, Postgres e Google Cloud Storage.

English

Codeflix is a complete streaming platform inspired by solutions such as Netflix and Amazon Prime Video. This project is part of the Full Cycle course and aims to practice advanced concepts such as distributed architecture, microservices, messaging, event-driven communication, centralized authentication, and scalability. The platform provides full video catalog management, including administration of categories, genres, and cast members, as well as an automated encoding pipeline responsible for converting videos to MPEG-DASH. The system is built using multiple independent services that communicate through HTTP, messaging, and event streaming, applying real-world concepts used in modern streaming platforms.

The admin catalog includes a React SPA frontend, a dedicated backend, and a MySQL database, enabling complete CRUD capabilities and RAW video upload. The encoding pipeline is implemented in Go, consumes events via RabbitMQ, and sends processed videos to Google Cloud Storage buckets. For the public catalog, there is a React frontend for subscribers and a public API, with Elasticsearch providing fast and efficient search, while Kafka, Kafka Connect, and Debezium handle real-time data replication. Subscription management is handled by a dedicated microservice using Postgres, and centralized authentication is performed through Keycloak with JWT-based secure access control. The architecture is documented using PlantUML with the C4 Model, representing users, system context, services, communication flow, databases, and brokers. The project leverages technologies such as React, Go, PHP(Laravel), Keycloak, RabbitMQ, Apache Kafka, Kafka Connect, Debezium, Elasticsearch, MySQL, Postgres, and Google Cloud Storage.
